# Claude Code 扩展开发中的关键概念

本文基于在 Claude Code 协作开发 `ask-anytime` 扩展过程中产生的术语解释，整理了 hook 机制、会话标识符管理等核心概念，并澄清了无关文本的识别方法。

## Hook（钩子）

### 定义
Hook 在本项目中特指 `write_session.js` 脚本，它是一个挂在 Claude Code 生命周期上的“钩子”程序。每次 Claude Code 启动时，系统会自动调用这个脚本。

### 作用
Hook 的核心功能是：**将当前 Claude Code 会话的 sessionId 保存到 `~/.claude/.ask_anytime_session.json` 文件中**。这样，`ask-anytime` 扩展就能通过读取该文件获知“当前是哪个 Claude Code 会话”，从而将用户的批注准确地导出到对应的知识文档中。

### 配置与常见问题
在项目的 `settings.json` 文件中，hook 需要按照 Claude Code 定义的规范格式配置。一个常见错误是：
- **旧版格式**：扁平结构（单层数组），Claude Code 更新 schema 后已不兼容。
- **正确格式**：需要双层数组 `hooks.SessionStart[i].hooks`。

调试路径通常是：先手动修改 `settings.json` 为正确格式，再修改扩展源码中的 `hookInstaller` 模块，确保新生成时采用正确格式，同时自动迁移旧的扁平条目。

### 与项目其他概念的关系
Hook 是实现会话持久化和跨会话知识管理的核心桥梁，它连接了 Claude Code 的自动启动机制和 `ask-anytime` 扩展的批注导出功能。没有 Hook，扩展就无法感知当前会话的上下文。

## SessionId（会话标识符）

（原文未明确单独定义，但通过 Hook 的解释可推导）

### 定义
SessionId 是 Claude Code 每次启动时生成的唯一会话标识符，用于区分不同的对话上下文。

### 用途
`ask-anytime` 扩展通过读取 `.ask_anytime_session.json` 文件中的 sessionId，确定“当前是哪个会话”，从而将用户的批注关联到正确的知识库条目。一般来说，每个会话对应一次独立的开发对话，sessionId 帮助扩展在多次启动之间维持上下文一致性。

### 与 Hook 的关联
Hook（`write_session.js`）负责在 Claude Code 启动时自动获取 sessionId 并写入文件，而扩展的其他模块则负责读取该文件并使用该标识符。两者构成了“写-读”的协同工作流。

## 无关文本与无关输入的处理

### 定义
在协作过程中，用户可能输入与当前项目无关的词汇（如“圣诞树”）或随机字符串（如 `hshdhsd`）。这些文本在项目上下文中没有实际含义，不代表代码实体或功能概念。

### 识别方法
- **无上下文**：如果某段文本在对话历史、项目代码、配置文件或讨论中从未出现，则无法基于项目上下文解释。
- **通识解释**：对于“圣诞树”这类通用名词，可给出与文化或常识相关的解释（如圣诞节的装饰树），但需明确指出与当前项目无关。
- **随机字符串**：如 `hshdhsd`，通常属于键盘乱输入、测试占位符或误操作，没有语义含义。

### 与项目的关系
这类文本与 `ask-anytime` 扩展、Claude Code 机制、CS-Script VSCode 扩展项目都没有直接关系——除非开发者明确将其用作代号、彩蛋或测试示例，否则应视为无关输入。
